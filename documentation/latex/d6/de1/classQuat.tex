\doxysubsection{Detailed Description}
Quaternion class, quaternions are 4 dimensional vectors which can describe rotations in 3 dimensional space if their length is 1.

They are written as\+:

$q = w + x \: i + y \: j + z \: k$

or in vector notation\+:

$q = [w, v] = [w, x, y, z]$

Where\+:

w = the real part v = the imaginary part, (x, y, z)

Note that we store the quaternion in a \doxylink{classVec4}{Vec4} as \mbox{[}x, y, z, w\mbox{]} because that makes it easy to extract the rotation axis of the quaternion\+:

q = \mbox{[}cos(angle / 2), sin(angle / 2) \texorpdfstring{$\ast$}{*} rotation\+\_\+axis\mbox{]}

Decomposes quaternion into swing and twist component\+:

$q = q_{swing} \: q_{twist}$

where $q_{swing} \: \hat{x} = q_{twist} \: \hat{y} = q_{twist} \: \hat{z} = 0$

In other words\+:


\begin{DoxyItemize}
\item $q_{twist}$ only rotates around the X-\/axis.
\item $q_{swing}$ only rotates around the Y and Z-\/axis.
\end{DoxyItemize}

Swing Twist Decomposition\+: any quaternion can be split up as\+:

\[q = q_{swing} \: q_{twist}\]

where $q_{twist}$ rotates only around axis v.

$q_{twist}$ is\+:

\[q_{twist} = \frac{[q_w, q_{ijk} \cdot v \: v]}{\left|[q_w, q_{ijk} \cdot v \: v]\right|}\]

where q\+\_\+w is the real part of the quaternion and q\+\_\+i the imaginary part (a 3 vector).

The swing can then be calculated as\+:

\[q_{swing} = q \: q_{twist}^* \]
