\doxysubsection{Detailed Description}
A rigid body that can be simulated using the physics system

Note that internally all properties (position, velocity etc.) are tracked relative to the center of mass of the object to simplify the simulation of the object.

The offset between the position of the body and the center of mass position of the body is \doxylink{classBody_a45950d9b3f8f28aafa13a81374912bf1}{Get\+Shape()}-\/\texorpdfstring{$>$}{>}Get\+Center\+Of\+Mass(). The functions that get/set the position of the body all indicate if they are relative to the center of mass or to the original position in which the shape was created.

The linear velocity is also velocity of the center of mass, to correct for this\+: $VelocityCOM = Velocity - AngularVelocity \times ShapeCOM$.

Change the body to a sensor. A sensor will receive collision callbacks, but will not cause any collision responses and can be used as a trigger volume. The cheapest sensor (in terms of CPU usage) is a sensor with motion type Static (they can be moved around using Body\+Interface\+::\+Set\+Position/\+Set\+Position\+And\+Rotation). These sensors will only detect collisions with active Dynamic or Kinematic bodies. As soon as a body go to sleep, the contact point with the sensor will be lost. If you make a sensor Dynamic or Kinematic and activate them, the sensor will be able to detect collisions with sleeping bodies too. An active sensor will never go to sleep automatically.
